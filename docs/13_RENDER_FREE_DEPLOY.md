# Render Free Deploy (Backend) + Neon + Vercel

This repo now includes a Render blueprint file at project root:
- `render.yaml`

And a startup script for backend deploys:
- `backend/render-start.sh`

## 1) Provision services

1. Create a Neon Postgres database (free tier).
2. Deploy frontend on Vercel (free tier) from `frontend/`.
3. Deploy backend on Render (free tier) from this repository using blueprint.

## 2) Deploy backend with Render blueprint

In Render:
1. New -> Blueprint
2. Connect this GitHub repository
3. Render reads `render.yaml` and creates `wealthmanagement-api`

## 3) Set backend environment variables in Render

Required:
- `DATABASE_URL` = Neon connection string
- `FRONTEND_URL` = Vercel app URL (e.g. `https://your-app.vercel.app`)
- `CORS_ORIGINS` = same URL as above (or comma-separated list)

Auto-generated by blueprint:
- `JWT_SECRET`

Optional:
- `GOOGLE_CLIENT_ID`
- `MICROSOFT_CLIENT_ID`

Already set in blueprint:
- `COOKIE_SECURE=true`

## 4) Startup behavior

On each deploy/start, Render runs:
1. `alembic upgrade head`
2. `uvicorn app.main:app --host 0.0.0.0 --port $PORT`

This is handled by `backend/render-start.sh`.

## 5) Frontend environment on Vercel

Set:
- `NEXT_PUBLIC_API_BASE_URL=https://<your-render-service>.onrender.com`

Optional for SSO:
- `NEXT_PUBLIC_GOOGLE_CLIENT_ID`
- `NEXT_PUBLIC_MICROSOFT_CLIENT_ID`
- `NEXT_PUBLIC_MICROSOFT_AUTHORITY`

## 6) Health check and smoke test

1. Open `https://<your-render-service>.onrender.com/health`
2. Expect response:

```json
{"status":"ok"}
```

3. Open frontend URL and verify API-backed pages load.

## 7) Notes

- Render free services can sleep when idle (cold start delays).
- Keep `DATABASE_URL` private and rotate if it was shared publicly.
- Example backend env template: `backend/.env.render.example`
